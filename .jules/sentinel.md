## 2024-05-28 — Unbounded JSON header & payload vulnerabilities in Wyoming STT integration
**Learning:** Wyoming STT implementations (`wakeword.py`, `wakeword_bench.py`) parse streaming text using a combination of line framing (`\n`) and explicit binary length markers (`data_length`, `payload_length`). The original client implementations trusted the size of incoming JSON metadata and payloads. An attacker could exploit this by sending unbounded metadata lines, large binary payloads, or fake payload lengths without actually sending payload bodies. The application reads into memory (and buffers arbitrarily large chunks) before returning results, exposing the tool to Memory Exhaustion (Denial-of-Service / OOM) attacks.
**Action:** Implemented safety bounds around parsing: `MAX_HEADER_BYTES = 65536` for inline newline parsing and `MAX_PAYLOAD_BYTES = 1048576` for nested/trailing chunk length parameters (`data_length` and `payload_length`) in `WakewordListener`, `WakewordActionListener`, and `_drain_detections`. Limits enforce a maximum size, rejecting out-of-bounds streaming operations aggressively with exceptions.
