
## 2024-05-18 — [Denial of Service via Unbounded Network Read in Wyoming Protocol]
**Learning:** Network protocols that frame headers with newlines and payloads with length fields are vulnerable to Denial of Service (DoS) attacks via unbounded memory allocation if the parser does not enforce strict limits on the number of bytes read before encountering a delimiter or allocating the payload buffer. The Wyoming protocol implementation in `wakeword.py` and `wakeword_bench.py` read byte-by-byte into a header line until `\n` without limit, and trusted `data_length` / `payload_length` fields to dynamically allocate memory up to an arbitrary size, allowing a malicious server to crash the application.
**Action:** Enforced a 64KB maximum limit on JSON header lines and a 1MB maximum limit on `data_length` and `payload_length` fields in `wakeword.py` and `wakeword_bench.py`. Additionally, `wakeword_bench.py`'s `_drain_detections` buffer was bounded to a maximum of 1MB + 64KB to prevent unbounded growth prior to newline detection.
