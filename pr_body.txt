💡 What
Added strict size bounds checks to the Wyoming protocol network parsers in `wakeword.py` and `wakeword_bench.py`.

🎯 Why
The `_drain_detections` and `read_loop` functions previously trusted the remote server to supply accurate `data_length` and `payload_length` fields, and safely terminate JSON framing with `\n`. A malformed stream (missing newlines, or claiming a 2GB payload length) could trigger an unbounded memory allocation in the Python socket handler, leading to an immediate out-of-memory crash (Denial of Service).

⚠️ Risk
Low. Legitimate payloads for wakeword chunks (typically fractions of a second of 16kHz s16le mono PCM) are nowhere near the enforced 1MB limit. Legitimate JSON header structures are small and far below the 64KB line read limit.

🔬 Verified
- Ran `pytest tests/` successfully; standard test payloads process seamlessly.
- Linted specific modified files via `ruff check` (no warnings introduced).
- Checked patches via `git diff --staged` to assure exact boundaries apply before unbounded appends.

📎 Scope
Network untrusted deserialization logic in daemon + bench. No drive-by refactoring.
